# doc-finder

Набір консольних утиліт для пошуку інформації в документах Microsoft Word (`.docx`): телефонів, e-mail адрес та довільного тексту з підтримкою підстановочних знаків.

## Можливості

**`phones`** — сканує всі `.docx` файли в папці та витягує:
- 📞 українські номери телефонів (у різних форматах: `+380...`, `0501234567`, `050-123-45-67` тощо)
- ✉️ електронні адреси

**`finder`** — шукає довільний текст у `.docx` файлах:
- підсвічує знайдені збіги червоним кольором прямо в терміналі
- підтримує пошук фраз (`"текст у лапках"`) та окремих слів
- якщо вказано кілька слів/фраз — виводить лише ті документи, де присутні **всі** вони одночасно
- підтримує підстановочні знаки `?` (один символ) та `*` (декілька символів)
- має режим `--paths-only` — виводить лише шляхи до знайдених файлів, зручно для скриптів (`mv`, `xargs`, `cp` тощо)

Обидві утиліти шукають не лише в основному тексті, а й у таблицях та колонтитулах (header/footer) документа.

## Вимоги

- Linux (скрипт встановлення розрахований на Bash)
- Python 3.8+
- Пакет [`python-docx`](https://pypi.org/project/python-docx/) (встановлюється автоматично при інсталяції)

## Встановлення

1. Склонуйте репозиторій:
   ```bash
   git clone https://github.com/devKalinchuk/doc_finder
   cd doc_finder
   ```

2. Запустіть скрипт встановлення:
   ```bash
   chmod +x install.sh
   ./install.sh
   ```

Скрипт `install.sh`:
- створює віртуальне середовище Python (`venv`) поруч із файлами проєкту та встановлює в нього `python-docx`;
- створює виконувані обгортки `phones` і `finder` у `~/.local/bin`, які запускають скрипти через це venv.

3. Переконайтесь, що `~/.local/bin` є у вашому `PATH`. Якщо команди `phones`/`finder` не знаходяться — додайте рядок нижче у `~/.bashrc` або `~/.profile`:
   ```bash
   export PATH="$HOME/.local/bin:$PATH"
   ```
   і виконайте:
   ```bash
   source ~/.bashrc   # або source ~/.profile
   ```

Після цього команди `phones` та `finder` доступні з будь-якої папки в терміналі.

## Використання: phones

```bash
phones [папка] [-r]
```

| Аргумент          | Опис                                                              |
|-------------------|--------------------------------------------------------------------|
| `папка`           | Шлях до папки з `.docx` файлами (за замовчуванням — поточна папка) |
| `-r`, `--recursive` | Шукати також у підпапках                                          |

**Приклади:**
```bash
phones                     # пошук телефонів та email у поточній папці
phones ~/Documents/Договори -r   # рекурсивний пошук у вказаній папці
```

## Використання: finder

```bash
finder [текст для пошуку ...] [-c] [-f ПАПКА] [-r] [-p]
```

| Аргумент               | Опис                                                                                          |
|------------------------|-------------------------------------------------------------------------------------------------|
| `текст для пошуку`     | Одне чи кілька слів/фраз. Фраза в лапках шукається як єдиний рядок. Кілька термінів — усі мають бути присутні в документі |
| `-c`, `--case-sensitive` | Точний пошук з урахуванням регістру (за замовчуванням регістр ігнорується)                      |
| `-f`, `--folder`       | Папка з `.docx` файлами (за замовчуванням — поточна папка)                                      |
| `-r`, `--recursive`    | Шукати також у підпапках                                                                        |
| `-p`, `--paths-only`   | Виводити лише абсолютні шляхи до знайдених файлів, без тексту та підсвічування                  |

**Підстановочні знаки:**
- `?` — будь-який один символ (наприклад, `тел?фон`)
- `*` — будь-яка кількість символів у межах слова (наприклад, `пошук*` знайде «пошук», «пошуковий»)

**Приклади:**
```bash
finder договір                          # знайти слово "договір" у поточній папці
finder "договір купівлі-продажу"        # знайти точну фразу
finder Іван телефон                     # документи, де є ОБИДВА слова
finder -r "акт виконаних робіт"         # пошук з підпапками
finder -c Договір                       # точний пошук з урахуванням регістру
```

**Приклад використання в скриптах** — перемістити всі знайдені договори купівлі-продажу в окрему папку:
```bash
mkdir -p contracts
finder -p "договір купівлі-продажу" | xargs -d '\n' -I{} mv {} contracts/
```
